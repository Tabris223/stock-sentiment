## 功能概述
实现基于 akshare 的东财个股新闻抓取功能，支持批量抓取、自动去重、数据库存储和 REST API 访问。

## 数据库设计

### news 表
```sql
CREATE TABLE news (
    id INTEGER PRIMARY KEY,
    stock_id INTEGER,           -- 关联股票
    title TEXT,                 -- 标题
    content TEXT,               -- 内容
    source TEXT,                -- 来源
    publish_time TIMESTAMP,     -- 发布时间
    url TEXT,                   -- 原始链接
    content_hash TEXT UNIQUE,   -- 去重用（SHA256）
    created_at TIMESTAMP
);
```

**设计决策**：
- 使用 SHA256(content|url) 作为去重哈希
- content_hash 设置 UNIQUE 约束，数据库层保证唯一性
- 外键关联 stocks 表
- publish_time 索引优化查询性能

## API 设计

### 1. POST /api/v1/news/fetch
手动触发新闻抓取

**Request**:
```json
{
  "stock_ids": [1, 2],  // 可选，为空则抓取所有
  "limit": 50           // 每只股票最多抓取数量
}
```

**Response**:
```json
{
  "fetched_count": 10,
  "saved_count": 10,
  "duplicates": 0,
  "message": "成功抓取 10 条新闻，保存 10 条新记录，跳过 0 条重复"
}
```

### 2. GET /api/v1/news
获取新闻列表

**Query Parameters**:
- `skip`: 跳过数量（分页）
- `limit`: 返回数量

**Response**:
```json
{
  "total": 10,
  "news": [...]
}
```

### 3. GET /api/v1/news/{stock_id}
获取指定股票的新闻

## 核心算法

### 去重算法
```python
def calculate_content_hash(title: str, url: str = "") -> str:
    """计算内容哈希用于去重"""
    content = f"{title}|{url}"
    return hashlib.sha256(content.encode('utf-8')).hexdigest()
```

**原理**：
1. 组合标题 + URL（URL 可能不同，但标题唯一性更强）
2. SHA256 哈希
3. 数据库 UNIQUE 约束保证唯一性
4. 先查询，重复则跳过

### 批量抓取流程
```
1. 获取股票列表（指定或全部）
2. 遍历每只股票
   a. 调用 akshare 抓取
   b. 遍历每条新闻
      i. 计算哈希
      ii. 查询是否已存在
      iii. 不存在则保存
3. 提交数据库事务
4. 返回统计信息
```

## 性能优化

1. **批量提交** - 所有新闻处理完后统一 commit，减少数据库事务次数
2. **索引优化** - publish_time、stock_id、content_hash 建立索引
3. **内存友好** - 逐条处理，不缓存所有新闻

## 错误处理

1. 股票不存在 → 404
2. 数据库为空 → 404
3. akshare 抓取失败 → 记录日志，继续处理其他股票
4. 数据验证失败 → 422
