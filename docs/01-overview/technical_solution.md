# Stock Sentiment - 技术方案

## 产品定位

**个人投资者舆情监控** —— 从"看新闻"到"读洞察"

让个人投资者拥有机构级的信息处理能力。

---

## 技术栈

| 层次 | 技术 | 说明 |
|------|------|------|
| 前端 | React + Vite + Tailwind | 响应式界面 |
| 后端 | FastAPI | Python 异步 API |
| 数据库 | SQLite | 轻量级持久化 |
| 定时任务 | APScheduler | 每 2 小时调度 |
| LLM | GLM5（智谱 API） | 舆情分析 + 投资建议 |
| Embedding | **Qwen3-Embedding-0.6B** | 新闻去重 |
| 推送 | 飞书 Webhook | 群消息推送 |

---

## 数据保留

| 数据类型 | 保留时间 |
|----------|----------|
| 原始新闻 | 14 天 |
| 分析结果 | 3 个月 |
| 聚合报告 | 6 个月 |

## 核心流程

```
每 2 小时执行：
┌─────────────────────────────────────────────────┐
│ 1. 新闻抓取（akshare → 东财个股新闻）             │
│    - 10 只股票 × ~20 条 = ~200 条               │
│    - 存入 news 表                               │
│                                                 │
│ 2. 向量去重（Qwen3-Embedding-0.6B）             │
│    - 与 3 天内新闻计算相似度                     │
│    - 阈值 > 0.85 标记为重复                      │
│    - 0 QPM 消耗（本地运行）                      │
│                                                 │
│ 3. 批量分析（GLM5）                             │
│    - 构建上下文文档（新新闻 + 历史摘要）          │
│    - 单次 LLM 调用完成：去重 + 分析 + 建议       │
│    - 输出结构化 JSON                            │
│                                                 │
│ 4. 持久化                                       │
│    - 分析结果 → analysis_results 表             │
│    - 投资建议 → investment_advice 表            │
│                                                 │
│ 5. 飞书推送                                     │
│    - 摘要 + 建议 → 飞书群                        │
└─────────────────────────────────────────────────┘

QPM 消耗：每 2 小时 1 次 = 0.008 QPM
```

## 时间权重设计

| 时间范围 | 权重 | 说明 |
|----------|:----:|------|
| 过去 2 小时 | 100% | 最高优先级 |
| 2-6 小时 | 70% | 今日早期 |
| 6-24 小时 | 40% | 昨日舆情 |
| 1-3 天 | 20% | 历史背景 |

---

## 投资建议风格

**偏向积极**：
- 中性新闻给 +2（默认假设"无消息=稳定"）
- 模糊情况给"持有"或"观望"，而非"减持"
- 优先关注机会，再提示风险

## 开发里程碑

| 阶段 | 交付物 | 工时 | 验收标准 |
|------|--------|:----:|----------|
| **P1: 数据管道** | 股票管理 + 定时抓取 + 存储 + API | 3 天 | 能注册 10 只股票，每 2 小时自动抓取入库 |
| **P2: 智能分析** | 去重 + GLM 批量分析 + 投资建议 | 3 天 | 能输出结构化分析，去重率 > 80% |
| **P3: 推送和前端** | 飞书推送 + Web 界面 | 3 天 | 用户能收到推送，Web 查看历史 |
| **P4: 稳定性** | 监控告警 + 错误处理 + 备份 | 2 天 | 失败能自动恢复并通知 |

**总工时**：11 人天（约 2 周）

---

## 风险与应对

| 风险 | 概率 | 应对措施 |
|------|:----:|----------|
| 东财反爬 | 中 | 随机 UA + 备用数据源（新浪） |
| GLM-5 不稳定 | 低 | 重试 3 次 + 失败跳过 |
| Mac 推理慢 | 低 | Qwen3-0.6B 够用，否则切云端 |
| 分析质量不稳定 | 中 | Prompt 迭代 + 人工抽检 |
