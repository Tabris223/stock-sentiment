## 老金需求

> 我们要把赛博舆情监控做成一个定期任务，我可以注册几只指定的股票，每天以小时级更新他们的信息。以供给Selene进行分析，分析结果进行持久化，有评分、有摘要和总结。需要能提供对投资有帮助的建议。

### 核心需求拆解

| 需求点 | 说明 |
|--------|------|
| 股票注册 | 用户可配置关注的股票列表（≤10只） |
| 定时更新 | **每 2 小时**抓取一次 |
| 舆情分析 | Selene 负责分析，**GLM5 批量处理** |
| 持久化 | 原始新闻 14 天 / 分析结果 3 个月 / 聚合报告 6 个月 |
| 评分 | 情绪评分、影响评分 |
| 摘要和总结 | 每只股票摘要 + 整体总结 |
| 投资建议 | **偏向积极**，基于分析给出操作建议 |
| 推送 | **飞书群** |
| 前端 | **需要 Web 界面** |

**技术约束**：
- QPM < 5
- GLM5 大上下文，取 30% 安全值
- 批量构建文档，一次处理多只股票

**🔴 老金补充需求（新闻去重）**：

> 定期抓取新闻时，不同数据源可能产生相同或类似的新闻，同一数据源滚动获取也可能重复。
> 
> **方案选择**：
> 1. 向量去重（embedding + 相似度）
> 2. 让 LLM（Selene）在分析时顺便做去重
> 
> **硬匹配不可行**，需要利用大语言模型能力。

**去重需求**：
- 跨数据源去重
- 时间窗口：同一天或三天内
- 方法：LLM 语义去重 或 向量相似度

### 第 3 轮：Selene（舆情分析师）✅

**Selene 输出**：

#### 新闻去重策略
- **时机**：抓取后、分析前
- **方法**：向量预过滤（0 QPM）+ LLM 批量语义去重（融入分析调用）
- **窗口**：3 天去重窗口 + 7 天引用窗口
- **QPM**：去重不单独调用，融入分析 Prompt

#### 舆情分析 Prompt
- 角色：专业舆情分析师，积极导向
- 输出：JSON 格式（去重结果 + 单股分析 + 投资建议）
- 偏向积极：中性新闻给 +2，模糊情况给"持有"

#### 摘要和总结
- 单股摘要：一句话 + 3 个要点 + 情绪趋势
- 多股总结：整体情绪 + 板块视图 + 行动建议
- 飞书格式：表格 + emoji + 快速链接

#### QPM 消耗
每 2 小时 1 次调用 = **0.008 QPM**（远低于 5 QPM 限制）

**🔴 老金补充（时间权重）**：
> 舆情时效性强，最新舆情权重更高
> 
> **权重设计**：
> - 过去 2 小时：100%（最高优先级）
> - 2-6 小时：70%
> - 6-24 小时：40%
> - 1-3 天：20%
> 
> 新消息与历史矛盾时，以新消息为准
